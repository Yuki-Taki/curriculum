■ ネットワーク層　
通信プロトコルの機能や役割を構成する層
単一のあるいは相互接続された複合的なネットワークの上で末端から末端までのデータを送り届ける役割を担うもの
•IP　Internet Protocol
相手先のアドレスの情報をもとにいてパケットを送信する機能
インターネット層のプロトコル、パケットを送受信するためのプロトコル
•IPヘッダ　
ＩＰにおけるデータの送受信単位であるＩＰパケットの先頭にある、一定の形式に従って制御情報などを記載した部分のこと。
送信元ＩＰアドレスや宛先ＩＰアドレスなど通信制御用のデータが記載されている。
•ICMP　Internet Control Message Protocol
IP通信の疎通状態を調査したり、正常に通信できない場合に送信元ホストへエラー情報を返したりする、IP通信を補佐するためのプロトコル
IPの通信にエラーがないが起こっていないか、起こった場合はどのようなエラーなのかというメッセージを提示する
•ICMPメッセージ
タイプ、コード、チェックサム、データの4つのフィールドにより構成。
２種類あり、１つは問い合わせ（Query）のメッセージ。あるノードから特定のノードに対する通信状態を確認することができる。
もう１つは、エラー通知（Error）のメッセージ。ノード間の通信で経路の途中でパケットが破棄されたりした場合に、その原因を送信元のノードにエラー通知する。
•IPアドレス
ネットワークに接続されたパソコンやプリンターなどに割り当てられる番号
 oオクテット
IPv4において８ビットずつ、カンマで区切られた部分を指す
 oネットワーク部
ネットワークを識別するアドレス
あるネットワークに割り当てられたIPアドレス群の中で、そのネットワーク自身を指し示すアドレス
 oホスト部
コンピュータや各機器を識別するアドレス
あるネットワークに割り当てられたIPアドレス群の中で、ネットワーク内の個々のホストに割り当てられるアドレス
 oクラス
ネットワーク部とホスト部に分かれる境界をIPv4アドレスの上位数ビットによって決定する方式
 oプライベートIPアドレス
インターネットプロバイダに割り振られているＩＰアドレスが、インターネットに接続するパソコンなどの機器に割り振られるＩＰアドレスのこと
 oグローバルIPアドレス
インターネットのようなグローバルなものではなく、社内ＬＡＮなど小規模なネットワークで使用されるＩＰアドレス
プライベート上のものになるため、インターネット上で使うことはできず、プライベートネットワークの中でのみ使われる。
•ネットワークアドレス
ホスト部の2進数のビットが全て０のアドレス、ネットワーク自体を指すアドレス
•ブロードキャストアドレス
ホスト部の2進数のビットが全て１のアドレス、あるセグメントに接続されている全てのホストにパケットを送信するためのアドレス
•ループバックアドレス
ネットワークアドレスとしても使用禁止されている「127.0.0.0/8」のネットワーク
自分自身を示す仮想的なIPアドレス
•サブネット
ネットワークの範囲
•サブネットマスク
ネットワークの範囲を指定するために必要な番号
 oプレフィックス表記
何ビット目までがネットワーク部かを簡易的にした表記
ＩＰアドレス記載に続いて、「/（ネットワーク部のビット数）」で表記
•デフォルトゲートウェイ
同一ネットワーク内のノードから外部ネットワークに属するノードと通信する際に出入り口となるデバイスやポートのこと
•IPv6
各コンピュータに割り振られるＩＰアドレスを１２８ビットのデータとして表現するもの
 oサブネットプレフィックス
IPv4におけるネットワーク部
基本的には前半６４ビットがサブネットプレフィックスにあたる
 oインターフェイスID
IPv4におけるホスト部
•ルータ
コンピュータネットワークにおいて、データを２つ以上の異なるネットワーク間に中継する通信機器
ネットワーク層において、どのルートを通してデータを転送すべきかを判断するルート選択機能を持つ
 oルーティングテーブル
ルータに記録される経路情報で、ルーティング処理を行う際に参照する
ネクストホップ
次に転送する最も近いルータやインターフェイスのこと
 oARPテーブル
ARPで割り出したIPアドレスとMACアドレスの対応表
ARP　Address Resolution Protocol
IPアドレスからイーサネットのMACアドレスの情報を得られるプロトコル
 oコアルータ
基幹ネットワークにおいてメインで用いられるルータ
 oセンタールータ
企業で中心的に利用されるルータ
 oエッジルータ
企業で利用されるルータ
 oブロードバンドルータ
家庭環境やＳＯＨＯ環境等、インターネット回線に接続することを目的として利用されるルータ
•ルーティング
異なるネットワークにパケットを送信する際に最適な経路を決めること
 o直接接続
ルータが直接接続しているネットワーク
 oスタティックルーティング
スタティックルートを使用したルーティングのこと
スタティックルートとは、管理者が宛先ネットワークへの最適なルートを手動で設定したルートのこと
 oダイナミックルーティング
ダイナミックルートとは、ルータで設定されたルーティングプロトコルで動的に追加されるルートのこと
ルータでルーティングプロトコルを使用したルーティングのことをダイナミックルーティングという
ディスタンスベクタ型
ホップ数（宛先に到着するまでに経由するデバイス数）をメトリックとし、経路を選ぶ方式
メトリックとは、ルーティングにおいて、最適経路を選択する際に指標となるもの
リンクステート型
各ルータが全体の接続情報を見て、最適経路を選択する方式
ハイブリッド型（拡張ディスタンスベクタ型）
ディスタンスベクタをベースに、リンクステート型を組み合わせ方式

■ トランスポート層
通信プロトコルの機能や役割を構成する層
データの送信元と送信先の間での制御や通知、交渉などを担うもの
•コネクション型通信　　　　　TCP
データ通信において、通信を開始する前に相手との間で仮想的な専門通信路（コネクション）を確立し、それを通じてデータの送受信を行う通信方式。伝送の信頼性や通信の品質が高い
 oACK　ACKnowledgement
二者間の通信で、相手方に何らかの肯定的な応答を返す際に送られる信号やデータ、パケットなどのこと
•コネクションレス型通信　　　　　UDP
相手との接続（コネクション）確立を行わず、データを送信する方式。通信を始める際の事前処理や通信中の状態確認を行わないため通信の信頼性は高くないが、通信コスト（負荷）が低く、遅延が少ない
•TCP　Transmission Control Protocol
インターネットなどのネットワークで、IPの一段階上移送のプロトコルとして標準的に使われるもの
コネクション型のプロトコルで、通信相手の状況を確認して接続を確立し、データの伝送が終わると切断する。相手が確実にデータを受け取ったか確認したり、データの欠落や破損を検知して再送したり、届いたデータを送信順に並べたりといった制御を行う。信頼性は高いが転送効率は低く、通信経路の品質が低いデータの到着に遅延が生じたり通信不能になりやすい。転送効率より確実性が重視される用途で利用される
 oスリーウェイハンドシェイク
IPネットワーク上でTCPによる接続（コネクション）を確立するための手順。パケットの送受信を3回行うことからこのように呼ばれる。
 oTelnet
インターネットなどのTCP/IPネットワークを通じて別のコンピュータにアクセスし、遠隔操作するためのプロトコル
 oHTTP　HyperText Transfer Protocol
WebサーバとWebクライアントの間でデータの送受信を行うために用いられるプロトコル
 oFTP　File Transfer Protocol
インターネットなどのTCP/IPネットワークでファイル転送を行うことができるプロトコル
 oSMTP　Simple Mail Protocol
インターネットなどのTCP/IPネットワークで、電子メールを伝達するためのプロトコル
•TCPヘッダ
TCPパケットの先頭部分にある、制御情報などを一定の形式で記述した領域。この後ろにデータ本体（ペイロード）を加えて、一つのＴＣＰパケットとなる。
•UDP
ＵＤＰはコネクションレス型のプロトコルで、通信相手が確実にデータを受け取ったかどうか確認したり、データの欠落を検知して再送したり、送信順と着信順を一致させるといった制御は行わず、データを送りっぱなしにする。信頼性は低いが転送効率が高く、遅延が発生しにくいため、多少のデータの欠落があっても高速性や即時性を重視する用途で利用される
 oRTP Real-time Transport Protocol
TCP/IPネットワーク上で音声や動画のように連続するデータの流れをリアルタイムに伝送するためのプロトコル
 oRTCP RTP Control Protocol
RTPなどを用いてデータをリアルタイム伝送する際に、送受信制御などを行うプロトコル
 oDHCP Dynamic Host Configuration Protocol
インターネットなどのIPネットワークに新たに接続した機器に、IPアドレスなど通信に必要な設定情報を自動的に割り当てるためのプロトコル
 oTFTP Trivial File Transfer Protocol
IPネットワーク上でファイル転送を行うためのプロトコル。ファイルの自動配布に用いられる
 oRIP Routing Information Protocol
IPネットワークで用いられるルーティングプロトコル。ルータなどの通信機器の間で経路情報を交換し、ある地点から別の地点までの最短経路を割り出すのに使われる
•UDPヘッダ
UDPヘッダ＋データ＝UDPデータグラム
・送信元ポート　通信相手からの応答を受付するポート番号。返答不要は０
・宛先ポート
通信相手のアプリケーションを示すポート番号。
・データ長
UDPヘッダとデータ部分を合わせたサイズ（バイト数）が入る
・チェックサム
UDPヘッダとデータ部分が正しいかチェックするために使用する。チェック不要の場合は０
